## Step 2 — Backend: KPI Computation Logic

**File:** `apps/backend/src/controllers/kpi.controller.ts`

Add a new `computeKpi` handler that aggregates live data into `kpi_metrics`:

### `computeKpi`
```
POST /api/kpi/compute
Auth: admin only
Body: { semester_id: number }
```

Computation steps (run in a transaction):

1. **Fetch the semester** record to get `starts_at` and `ends_at`.
2. **Delete old computed rows** for this semester:
   ```sql
   DELETE FROM kpi_metrics WHERE semester_id = ? AND metric_key IN ('attendance_count','achievement_count','member_count')
   ```
3. **Attendance count per club:**
   ```sql
   SELECT e.club_id, COUNT(a.id) AS cnt
   FROM attendance a
   JOIN events e ON e.id = a.event_id
   WHERE e.status = 'published'
     AND a.checked_in_at BETWEEN ? AND ?
   GROUP BY e.club_id
   ```
   Insert as `metric_key = 'attendance_count'`.

4. **Achievement count per club:**
   ```sql
   SELECT club_id, COUNT(id) AS cnt
   FROM achievements
   WHERE awarded_at BETWEEN ? AND ?
   GROUP BY club_id
   ```
   Insert as `metric_key = 'achievement_count'`.

5. **Active member count per club** (if FR-005 is implemented):
   ```sql
   SELECT club_id, COUNT(*) AS cnt
   FROM memberships WHERE status = 'active'
   GROUP BY club_id
   ```
   Insert as `metric_key = 'member_count'`.

6. **Total score** = attendance_count + achievement_count (configurable weights can be added later).
   Insert as `metric_key = 'total_score'`.

Return `{ computed: true, semester_id, clubs_updated: N }`.

## Step 5 — Backend: Leaderboard Tied Ranks

**File:** `apps/backend/src/models/kpi.model.ts`

Current query uses raw `ORDER BY total_score DESC`. Fix tied ranks in application code:

```ts
let rank = 1;
const ranked = rows.map((row, i) => {
  if (i > 0 && row.total_score < rows[i - 1].total_score) rank = i + 1;
  return { ...row, rank };
});
```
