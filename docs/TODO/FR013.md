# FR-013 — Daily MCQ Questions Created by Club Leaders

**Priority:** High
**Status:** Planned (the app has clubs, events, achievements, attendance, and student engagement surfaces, but it does not yet support club-created daily learning questions)

---

## Summary

Add a daily multiple-choice question feature where each club leader can create simple educational MCQs for students.

Each question should belong to a club and include:

- Question text
- Exactly 4 answer choices
- One correct answer
- Optional explanation
- XP reward settings

Students can answer one or more daily questions and receive:

- XP for participation
- Extra XP for correct answers
- Daily streak progress when they keep answering consistently

Examples:

- Cybersecurity Club: `What does VPN stand for?`
- Robotics Club: `Which sensor is used to measure distance?`
- Literature Club: `Who wrote Romeo and Juliet?`

This feature keeps the platform active every day, gives clubs a lightweight educational tool, and adds fresh content created directly by club leaders.

---

## Role Mapping

The current codebase uses the role name `club_leader`, not `club president`.

For FR013:

- `club leader` = the person allowed to create and manage daily MCQs for their club
- `admin` = can manage all clubs’ daily MCQs
- `student` = can answer published daily MCQs

---

## Goal

Create a daily engagement loop that encourages students to open the platform every day and learn something useful from clubs.

The feature should provide:

- A daily question feed for students
- A simple creation/management panel for club leaders
- XP rewards integrated with the gamification system from FR012
- A visible answer streak that motivates repeat participation

---

## V1 Product Scope

### In scope

- Club leaders create daily MCQs for clubs they manage
- Students can answer each published question once
- Students may answer multiple questions on the same day
- Students earn participation XP and correct-answer bonus XP
- Students see whether they were correct and can read the explanation after answering
- The system tracks a daily answer streak
- Admins can view and manage all clubs’ daily questions

### Out of scope for V1

- Timed quizzes
- Question banks or AI-generated questions
- Rich media questions (image/audio/video)
- Negative marking
- Team-based quiz competitions
- Push notifications for every new question

---

## Product Rules

### Publishing rules

- Questions are created in `draft` first
- Only `published` questions appear to students
- A question belongs to one club
- Multiple questions may be published on the same date
- Daily availability must use one consistent app timezone, not browser-local guesses
- reuse one shared backend date/time utility or config value for:
  - active question dates
  - streak calculations
  - FR012 daily login XP

### Student answering rules

- A student can answer each question only once
- Students can answer questions from any published club in V1
- Correct answer and explanation are hidden until the student submits an answer
- After answering, the result is locked and cannot be changed

### Content language rule

The current codebase stores clubs and events in bilingual fields, but that pattern does not exist yet for daily questions.

Recommended V1 decision:

- store one authored language per question using the simple fields in this plan:
  - `question_text`
  - `explanation`
  - `option_text`
- club leaders choose the language of the question content
- app navigation and chrome remain translated through the normal i18n files

Future enhancement:

- if the product later requires bilingual authoring, add explicit `*_ar` or equivalent parallel fields in a follow-up instead of quietly implying that support in V1

### XP rules

Each question should support two XP values:

- `participation_xp`
- `correct_bonus_xp`

Recommended defaults:

| Field | Default |
|---|---:|
| `participation_xp` | 5 |
| `correct_bonus_xp` | 10 |

Recommended validation caps:

- `participation_xp`: `0-10`
- `correct_bonus_xp`: `0-20`

This prevents club leaders from creating overpowered questions that distort the platform economy.

### Streak rule

A student’s daily-question streak increases when they answer at least one published daily question on a valid question day.

For V1:

- answering 3 questions on the same day still counts as one streak day
- skipping a valid question day breaks the streak
- `best_streak` should be stored as well as `current_streak`

## Step 1 — Data Model and Migration

Create the next migration after FR012, for example:

**File:** `apps/backend/migrations/026_create_daily_questions.sql`

### Create `daily_questions`

```sql
CREATE TABLE daily_questions (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  club_id INTEGER NOT NULL,
  question_text TEXT NOT NULL,
  explanation TEXT NULL,
  active_date TEXT NOT NULL,
  participation_xp INTEGER NOT NULL DEFAULT 5,
  correct_bonus_xp INTEGER NOT NULL DEFAULT 10,
  status TEXT NOT NULL DEFAULT 'draft',
  created_by INTEGER NOT NULL,
  published_at TEXT NULL,
  created_at TEXT NOT NULL DEFAULT (datetime('now')),
  updated_at TEXT NOT NULL DEFAULT (datetime('now')),
  FOREIGN KEY (club_id) REFERENCES clubs(id) ON DELETE CASCADE,
  FOREIGN KEY (created_by) REFERENCES users(id) ON DELETE CASCADE
);
```

### Create `daily_question_options`

```sql
CREATE TABLE daily_question_options (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  question_id INTEGER NOT NULL,
  option_key TEXT NOT NULL,
  option_text TEXT NOT NULL,
  sort_order INTEGER NOT NULL,
  is_correct INTEGER NOT NULL DEFAULT 0,
  FOREIGN KEY (question_id) REFERENCES daily_questions(id) ON DELETE CASCADE,
  UNIQUE(question_id, option_key)
);
```

App-level validation should enforce:

- exactly 4 options
- exactly 1 correct option

### Create `daily_question_answers`

```sql
CREATE TABLE daily_question_answers (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  question_id INTEGER NOT NULL,
  user_id INTEGER NOT NULL,
  selected_option_id INTEGER NOT NULL,
  is_correct INTEGER NOT NULL,
  participation_xp_awarded INTEGER NOT NULL DEFAULT 0,
  correct_bonus_xp_awarded INTEGER NOT NULL DEFAULT 0,
  answered_on TEXT NOT NULL,
  answered_at TEXT NOT NULL DEFAULT (datetime('now')),
  FOREIGN KEY (question_id) REFERENCES daily_questions(id) ON DELETE CASCADE,
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
  FOREIGN KEY (selected_option_id) REFERENCES daily_question_options(id) ON DELETE CASCADE,
  UNIQUE(question_id, user_id)
);
```

### Create `daily_question_streaks`

```sql
CREATE TABLE daily_question_streaks (
  user_id INTEGER PRIMARY KEY,
  current_streak INTEGER NOT NULL DEFAULT 0,
  best_streak INTEGER NOT NULL DEFAULT 0,
  last_answered_on TEXT NULL,
  updated_at TEXT NOT NULL DEFAULT (datetime('now')),
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);
```

## Step 3 — XP and Gamification Integration

FR013 should reuse the gamification service planned in FR012.

Recommended XP events:

- `daily_question_participation`
- `daily_question_correct_bonus`

Recommended deterministic reference keys:

- `daily-question:QUESTION_ID:USER_ID:participation`
- `daily-question:QUESTION_ID:USER_ID:correct`

On answer submission:

1. Insert the answer once
2. Award participation XP once
3. If correct, award correct-bonus XP once
4. Update streak once for the answer day
5. Return the updated XP/level summary when available

This keeps FR013 aligned with the same anti-duplication model as FR012.

## Step 7 — Streak Logic

Streak progress should be visible to students and returned by the API.

Recommended summary shape:

```json
{
  "current_streak": 4,
  "best_streak": 9,
  "last_answered_on": "2026-04-16"
}
```

Recommended logic:

- first valid answer day -> streak becomes `1`
- consecutive valid answer day -> `current_streak + 1`
- gap in valid answer days -> streak resets to `1`
- `best_streak` updates when `current_streak` exceeds it

If the product team wants maximum fairness, a “valid question day” should mean a date where at least one published daily question existed in the student feed.

## Assumptions

- FR013 should reuse the XP/level system from FR012 rather than inventing a separate reward engine.
- In V1, students can answer published questions from any club, not only clubs they joined.
- The current role `club_leader` is the correct implementation target for the user’s “club president” idea.
- In V1, daily-question content itself is single-authored-language content; bilingual authoring is a follow-up feature, not an implied requirement of this schema.
- FR013 should share one app-timezone/date rule with FR012 so question availability and streaks use the same calendar semantics everywhere.
- Daily questions are lightweight engagement content, not high-stakes exams, so immediate educational feedback after answer submission is acceptable.
